# Fix Next.js TypeScript Types in Docker

## Problem Statement

Docker web image build was failing due to incorrect file references in `Dockerfile.web`. During investigation, an unnecessary path was added to `tsconfig.json`.

## Root Cause Analysis

1. **Original issue**: `Dockerfile.web` referenced `apps/web/next-env.d.ts` which doesn't exist - this file is auto-generated by Next.js at build time

2. **Incorrect fix**: Someone added `.next/dev/dev/types/**/*.ts` to tsconfig.json which doesn't actually exist in the filesystem

3. **Actual type paths that exist**:
   - `.next/types/` - standard Next.js generated types
   - `.next/dev/types/` - Turbopack/Next.js dev server generated types

## Fix Applied

**Dockerfile.web** - Removed non-existent file reference:
```diff
- COPY apps/web/tsconfig.json apps/web/next.config.js apps/web/next-env.d.ts apps/web/vitest.config.ts apps/web/vitest.setup.ts ./apps/web/
+ COPY apps/web/tsconfig.json apps/web/next.config.js apps/web/vitest.config.ts apps/web/vitest.setup.ts ./apps/web/
```

**tsconfig.json** - Removed unnecessary path:
```diff
  "include": [
    "next-env.d.ts",
    "**/*.ts",
    "**/*.tsx",
    ".next/types/**/*.ts",
-   ".next/dev/dev/types/**/*.ts"
  ]
```

## Verification

- [x] Docker build succeeds: `docker build -f docker/Dockerfile.web --target dev .`
- [x] Docker compose starts: `docker compose up -d`
- [x] Unit tests pass: `npm run docker:test`
- [x] E2E tests pass: `npm run docker:test:e2e` (113/115)

## Notes

- The `next-env.d.ts` file is auto-generated by Next.js during build and should never be copied in Dockerfile
- The original tsconfig.json was correct - only needed the Dockerfile fix
- The `.next/dev/dev/types/` path was a misdiagnosis and doesn't exist
