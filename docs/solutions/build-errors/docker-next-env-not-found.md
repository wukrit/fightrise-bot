# Docker Web Image Build Failure - next-env.d.ts Not Found

## Problem

Docker web image build was failing with error:
```
"/apps/web/next-env.d.ts": not found
```

## Root Cause

The `Dockerfile.web` referenced `apps/web/next-env.d.ts` which doesn't exist in the source tree. This file is auto-generated by Next.js during build time and should never be copied in the Dockerfile.

## Investigation

1. Initial fix attempt incorrectly added `.next/dev/dev/types/**/*.ts` to tsconfig.json
2. This path doesn't actually exist in the filesystem
3. Real type paths that exist:
   - `.next/types/` - standard Next.js types
   - `.next/dev/types/` - Turbopack dev types

## Solution

**Dockerfile.web** - Remove non-existent file reference:
```dockerfile
# Before
COPY apps/web/tsconfig.json apps/web/next.config.js apps/web/next-env.d.ts apps/web/vitest.config.ts apps/web/vitest.setup.ts ./apps/web/

# After
COPY apps/web/tsconfig.json apps/web/next.config.js apps/web/vitest.config.ts apps/web/vitest.setup.ts ./apps/web/
```

**tsconfig.json** - The original was correct, removed incorrect fix:
```json
"include": [
  "next-env.d.ts",
  "**/*.ts",
  "**/*.tsx",
  ".next/types/**/*.ts",
  ".next/dev/types/**/*.ts"
]
```

## Prevention

- Never reference auto-generated files in Dockerfiles
- `next-env.d.ts` is created by Next.js at build time
- Similar files to never copy: `.next/`, `node_modules/.cache/`
