# Plan: Fix Audit Page Tournament Filtering

## Task: Add tournamentId filtering to server-side audit page

**File:** `apps/web/app/tournaments/[id]/admin/audit/page.tsx`

### Problem
The server-side `getAuditLogs` function (lines 25-31) only filters by `action`, not by `tournamentId`. This is a security issue - it exposes audit logs from ALL tournaments to any admin.

### Solution
Add tournamentId filtering using the same pattern as the API route:
1. Filter by `entityType: 'Registration'`
2. Filter by `entityId` being in the tournament's registration IDs

### Implementation Steps

1. **Add tournamentId to the where clause**
   - Import `registration` from prisma
   - Modify the `where` clause to filter audit logs by registration IDs that belong to the tournament

2. **Code changes** (in `getAuditLogs` function):

   ```typescript
   // Current (lines 25-27):
   const where: Record<string, unknown> = {
     action: { in: REGISTRATION_ACTIONS },
   };

   // Fixed:
   const where: Record<string, unknown> = {
     action: { in: REGISTRATION_ACTIONS },
     entityType: 'Registration',
     entityId: {
       in: await prisma.registration
         .findMany({
           where: { tournamentId },
           select: { id: true },
         })
         .then((regs) => regs.map((r) => r.id)),
     },
   };
   ```

3. **Verify** the fix matches the API route behavior at `apps/web/app/api/tournaments/[id]/admin/audit/route.ts` (lines 88-93)

## Verification

- [ ] Server-side audit page only returns logs for the specified tournament
- [ ] No audit logs from other tournaments are exposed
- [ ] Filtering matches the API route behavior
