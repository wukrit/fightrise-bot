# Requirements: FightRise Admin Web Portal

**Defined:** 2026-02-25
**Core Value:** Run Start.gg tournaments entirely within Discord — now with web-based admin tools.

## v1 Requirements

### Authentication & Authorization

- [x] **AUTH-01**: User can sign in via Discord OAuth
- [x] **AUTH-02**: User can access tournament admin pages only if they have TournamentAdmin role in database
- [x] **AUTH-03**: API endpoints verify tournament admin role before returning data
- [x] **AUTH-04**: Unauthorized access returns 403 instead of exposing data

### Admin API

- [x] **API-01**: GET /api/tournaments/[id]/admin/registrations - List registrations with filters
- [x] **API-02**: POST /api/tournaments/[id]/admin/registrations - Create manual registration
- [x] **API-03**: PATCH /api/tournaments/[id]/admin/registrations/[id] - Update registration status
- [x] **API-04**: DELETE /api/tournaments/[id]/admin/registrations/[id] - Remove registration
- [x] **API-05**: GET /api/tournaments/[id]/admin/matches - List matches with filters
- [x] **API-06**: POST /api/tournaments/[id]/admin/players/[id]/dq - Disqualify player
- [x] **API-07**: GET /api/tournaments/[id]/admin/audit - List audit logs

### Admin UI - Dashboard

- [x] **DASH-01**: Admin can view tournament dashboard at /tournaments/[id]/admin
- [x] **DASH-02**: Dashboard shows tournament state, entrant count, match count
- [x] **DASH-03**: Dashboard shows recent admin actions

### Admin UI - Registrations

- [x] **REG-01**: Admin can view all registrations in a table with status filters
- [x] **REG-02**: Admin can approve pending registrations
- [x] **REG-03**: Admin can reject registrations with reason
- [x] **REG-04**: Admin can manually register a player (walk-in)
- [x] **REG-05**: Admin can remove a registration
- [x] **REG-06**: Registration table supports pagination

### Admin UI - Matches

- [x] **MATCH-01**: Admin can view all matches in a table with filters (round, state)
- [x] **MATCH-02**: Admin can view match details (players, scores)
- [x] **MATCH-03**: Admin can view check-in status for matches

### Admin UI - Disqualification

- [x] **DQ-01**: Admin can disqualify a player from a match
- [x] **DQ-02**: DQ form requires reason input
- [x] **DQ-03**: DQ action creates audit log entry
- [x] **DQ-04**: DQ syncs to Start.gg via API

### Admin UI - Audit Logs

- [x] **AUDIT-01**: Admin can view audit log for tournament
- [x] **AUDIT-02**: Audit log shows action, user, timestamp, details
- [x] **AUDIT-03**: Audit log supports filtering by action type

## v2 Requirements

### Advanced Features

- **SEED-01**: Admin can reorder seeding via drag-drop
- **CHECK-01**: Admin can view real-time check-in dashboard
- **CHECK-02**: Admin can send check-in reminders
- **STATE-01**: Admin can override match state (emergency)
- **BRACKET-01**: Embed Start.gg bracket viewer

### Export

- **EXP-01**: Export registrations to CSV
- **EXP-02**: Export match results to CSV

## Out of Scope

| Feature | Reason |
|---------|--------|
| Tournament creation from web | Use Discord bot for v1 |
| Full bracket editor | Sync issues with Start.gg - embed instead |
| Bulk messaging | Rate limiting complexity |
| Multi-event management | Defer to v2 |
