## Integration Issues

### Issue 1: Audit Page Server-Side Not Filtering by Tournament

| Attribute | Value |
|-----------|-------|
| **Severity** | Medium |
| **Phase** | Phase 2: Registration Management |
| **Affected Requirements** | AUDIT-01, AUDIT-02, AUDIT-03 |
| **File** | `apps/web/app/tournaments/[id]/admin/audit/page.tsx` |

**Problem:** The server-side audit page receives `tournamentId` as a parameter but does NOT use it to filter audit logs. This could leak audit logs from other tournaments to admins.

**Evidence:**
```typescript
// Lines 25-31 in audit/page.tsx
const where: Record<string, unknown> = {
  action: { in: REGISTRATION_ACTIONS },
};
// tournamentId is NOT used here
```

**Contrast with working API route** (lines 88-97 in `/api/tournaments/[id]/admin/audit/route.ts`):
```typescript
const where: Prisma.AuditLogWhereInput = {
  action: { in: actions },
  OR: [
    { entityType: 'Registration', entityId: { in: regIds } },
    { userId: { in: adminUserIds } },
  ],
};
```

**Fix Required:** Add tournamentId filtering to the audit logs query in the server-side page, similar to how the API route does it.
